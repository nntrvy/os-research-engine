## How We Manage Work

We keep things simple. Two systems:

| System | Purpose | Tool |
|--------|---------|------|
| **To-dos** | Non-critical, routine tasks | Checklist / Notion |
| **Card Table** | Important tasks, projects, experiments | Kanban board |

---

## To-dos (Non-Critical Tasks)

**What goes here:**
- Small tasks that take <2 hours
- Routine work (admin, updates, follow-ups)
- Personal reminders
- Quick fixes

**How it works:**

```
□ Task description — @owner — due date (optional)
```

**Examples:**
```
□ Send weekly report to founder — @you — Friday
□ Update experiment log for Project X — @you
□ Reply to vendor email — @you — Today
□ Review teammate's draft — @you — Tomorrow
```

**Rules:**
1. If it takes <5 minutes, do it now. Don't add to the list.
2. Check your To-dos every morning.
3. Clear completed items daily.
4. If a task keeps getting pushed, either do it or delete it.

---

## Card Table (Important Tasks)

**What goes here:**
- Tasks tied to OKRs or key projects
- Experiment work (research, build, test)
- Cross-functional collaboration
- Anything that takes >2 hours or involves others

**How it works:**

We use a **Kanban board** with 4 columns:

| Backlog | In Progress | Review | Done |
|---------|-------------|--------|------|
| Tasks waiting to start | Currently being worked on | Needs feedback/approval | Completed |

**Card format:**

```
┌─────────────────────────────────────┐
│ [PROJECT TAG]                       │
│ Card Title                          │
│─────────────────────────────────────│
│ Owner: @name                        │
│ Due: DD/MM                          │
│ Priority: 🔴 High / 🟡 Medium / 🟢 Low │
│─────────────────────────────────────│
│ Description:                        │
│ Brief context + expected outcome    │
│─────────────────────────────────────│
│ Checklist:                          │
│ □ Subtask 1                         │
│ □ Subtask 2                         │
│ □ Subtask 3                         │
└─────────────────────────────────────┘
```

**Example Card:**

```
┌─────────────────────────────────────┐
│ [EXPERIMENT]                        │
│ Run fake door ad for Project Alpha  │
│─────────────────────────────────────│
│ Owner: @minh                        │
│ Due: 15/03                          │
│ Priority: 🔴 High                    │
│─────────────────────────────────────│
│ Description:                        │
│ Test demand for Alpha concept using │
│ Meta ads → landing page → signup.   │
│ Target: 1000 impressions, 50 clicks │
│─────────────────────────────────────│
│ Checklist:                          │
│ □ Create ad creative (2 versions)   │
│ □ Set up landing page               │
│ □ Configure tracking (UTM + pixel)  │
│ □ Launch campaign                   │
│ □ Document results in experiment log│
└─────────────────────────────────────┘
```

---

## Rules of the Card Table

1. **One owner per card.** No shared ownership. One person is accountable.
2. **Max 3 cards "In Progress" per person.** Focus beats multitasking.
3. **Update status daily.** Move cards, add comments, flag blockers.
4. **Blocked? Escalate fast.** Don't let cards sit. Ask for help.
5. **Done = documented.** A task isn't done until the outcome is recorded.
