# Change Log

This folder tracks all work sessions and changes to the OS Research Engine repository.

## Structure

```
change-log/
├── README.md           # This file
├── CHANGELOG.md        # Master change tracker (concise entries)
└── sessions/           # Detailed session summaries
    ├── 03.12.25-session-summary.md
    ├── 04.12.25-session-summary.md
    ├── 08.12.25-session-summary.md
    ├── 25.01.26-session-summary.md
    └── 01.02.26-session-summary.md
```

## How to Use

### Finding Changes

1. **Quick lookup**: Check `CHANGELOG.md` for one-line entries organized by date
2. **Deep dive**: Open the corresponding session file in `sessions/` for full context

### Adding New Sessions

1. Create `sessions/DD.MM.YY-session-summary.md` with full details
2. Add concise entries to `CHANGELOG.md` under the date header

### Session Summary Template

Each session summary should include:
- **Session Overview**: Duration, focus, key outcome
- **Findings & Discoveries**: Market insights, data collected
- **Documents Modified**: Table of files changed
- **Decisions Made**: Strategic choices with rationale
- **Open Questions**: Unresolved items for future sessions
- **Next Steps**: Actionable items for follow-up

## Naming Convention

Files use `DD.MM.YY` format (e.g., `01.02.26` = February 1, 2026).
