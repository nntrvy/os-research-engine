## Purpose

Build the absolute minimum product with only one core feature to test your most critical assumption.

**âœ… Ideal for:** Validating your core value proposition before building a full product

## Preparation

1. **Identify your single most important feature**
   - What is the ONE thing that creates value?
   - What's the riskiest assumption?
2. **Strip away everything else**
   - No nice-to-haves
   - No future features
   - No polish
3. **Define success criteria**
4. **Build only what's necessary** for that one feature

## Execution

1. **Launch to a small group** of target users
2. **Measure usage** of the core feature
3. **Collect feedback** on whether it solves their problem
4. **Track key metrics** defined in success criteria

## Analysis

- Do users use the core feature?
- Does it solve their problem?
- Would they pay for it?
- What's missing that would make them pay more?
- Should we continue building?

## "Is It Ready?" Checklist

Before launching your Single Feature MVP:

- [ ] Does it deliver the core value?
- [ ] Can users complete the main job?
- [ ] Can you measure success?
- [ ] Is it stable enough for testing?
- [ ] Can you onboard users quickly?

## Best Practices

### âœ… Do:
- Be ruthless about cutting features
- Talk to users constantly
- Measure one key metric obsessively
- Ship faster than comfortable
- Learn, don't prove

### âŒ Don't:
- Add "just one more feature"
- Wait until it's polished
- Launch to too many people
- Ignore negative feedback
- Build before validating
